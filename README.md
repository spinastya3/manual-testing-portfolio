# 💼 Портфолио: Ручное тестирование (Manual Testing)

В данном репозитории представлены примеры моей работы над тестовой документацией в рамках обучения в Яндекс Практикуме. 
---

### 📋 Проект 1: Веб-сервис и мобильное приложение Яндекс.Самокат (Дипломный проект)
**Суть:** Комплексное функциональное, UI и кроссплатформенное тестирование веб-интерфейса и мобильного приложения (Android-сборки для курьеров) во взаимодействии с API и БД.

*   **Управление дефектами (74 баг-репорта):** Найдено, локализовано и детально задокументировано 74 дефекта различного уровня критичности (от локальных багов верстки до блокирующих ошибок бэкенда и падений UI). Все отчеты оформлены по стандартам QA (Severity, STR, AR/ER, Environment).
*   **Мобильное и интеграционное тестирование:** Проектирование тестов для Android-приложения (эмулятор Android SDK). Реализация сквозных сценариев с использованием REST API (`POST /api/v1/orders`, `POST /api/v1/courier`) для генерации предусловий и манипуляциями с базой данных (PostgreSQL) для проверки граничных состояний заказов.
*   **Тестирование системных функций ОС:** Покрытие проверками системных PUSH-уведомлений, логики работы приложения в бэкграунде (свернуто/закрыто), а также обработки прерываний связи (Авиарежим, потеря Wi-Fi/Cellular сети, восстановление коннекта).
*   **Применение тест-дизайна:** Проектирование тестов с использованием техник *эквивалентного разделения (EP)* и *анализа граничных значений (BVA)* для текстовых полей веб-формы (Имя, Фамилия, Адрес, Номер телефона).
*   **Исследовательское тестирование (Exploratory Testing):** Свободный поиск дефектов за рамками документации. Локализованы критические баги: падение сайта (500 Error) на клавиатурных событиях и блокировка бизнес-логики UI.
*   **Кроссбраузерность и адаптивность:** Проведение проверок отображения элементов интерфейса в браузерах *Google Chrome* и *Яндекс.Браузер* на разрешениях *1920×1080* и *1280×720*.

### 📋 Проект 2: API Яндекс.Прилавок
**Суть:** Комплексное тестирование бэкенд-части сервиса заказов продуктов.

*   **Анализ API-документации:** Работа с требованиями и спецификацией в формате Swagger (Apidoc).
*   **Тест-дизайн:** Разработка чек-листов для проверки различных HTTP-методов (`GET`, `POST`, `PUT`, `DELETE`).
*   **Тестирование логики:** Проверка статус-кодов, структуры JSON-ответов и валидации обязательных/необязательных параметров.
*   **Баг-репортинг:** Фиксация дефектов, найденных в ходе выполнения интеграционных и позитивных/негативных проверок.

### 📋 Проект 3: Мобильное приложение Яндекс.Метро (Android)
**Суть:** Тестирование новой сборки после рефакторинга и проведение полного регрессионного цикла.

*   **Функциональное тестирование:** Проверка обновленной логики построения маршрутов приложения согласно выделенным требованиям.
*   **Регрессионное тестирование:** Контроль стабильности текущих функций после внесения правок разработчиками в код.
*   **Mobile Specific Testing:** Тестирование особенностей мобильной среды (переключение сетей 3G/LTE/Wi-Fi, работа в авиарежиме, входящие звонки, смена ориентации экрана, прерывания).
*   **Тестирование на эмуляторе:** Работа в Android Studio с конфигурацией виртуального устройства (Honor 8, Android 9.0).
*   **Баг-репортинг:** Составление детальных отчетов о дефектах с указанием специфики мобильного окружения.

### 📋 Проект 4: Веб-сервис Яндекс.Маршруты (Функционал каршеринга)
**Суть:** Тестирование нового модуля бронирования автомобилей: от проверки верстки по макетам до логики оплаты.

*   **Тестирование верстки (UI):** Проверка соответствия пиксельным макетам формы бронирования и интерактивных элементов карты.
*   **Тест-дизайн сложных форм:** Составление чек-листов для логики модальных окон «Способ оплаты» и «Добавление карты».
*   **Функциональное тестирование:** Написание детальных тест-кейсов для критического пути пользователя (кнопка «Забронировать»).
*   **Кроссбраузерное тестирование:** Проверка корректности отображения элементов в разных независимых браузерах.
*   **Баг-репортинг:** Фиксация дефектов интерфейса и логики с подробными шагами воспроизведения (STR).
