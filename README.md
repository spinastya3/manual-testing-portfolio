# Портфолио: Ручное тестирование (Manual Testing)

В данном репозитории представлены примеры моей работы над тестовой документацией в рамках обучения в Яндекс Практикуме.

## 📋 Проект 1: API Яндекс.Прилавок
Комплексное тестирование бэкенд-части сервиса заказов продуктов.

### Выполненные задачи:
* **Анализ API-документации:** работа с требованиями в формате Swagger (Apidoc).
* **Тест-дизайн:** разработка чек-листов для проверки различных HTTP-методов (GET, POST, PUT, DELETE).
* **Тестирование логики:** проверка статус-кодов, структуры JSON-ответов и валидации параметров.
* **Баг-репортинг:** фиксация дефектов, найденных в ходе выполнения проверок.

### Ссылки на артефакты:
* 📊 [Google Таблица: Чек-лист и результаты тестов](https://docs.google.com/spreadsheets/d/1LmHhenwbE_nQMpJdR0_0InDabNXxmbP8VA0nT0xydAA/edit?gid=2006427015#gid=2006427015) — более 100 проверок API (валидация параметров, статус-коды, структура JSON-ответов).

*Документация подготовлена с соблюдением принципов полноты, атомарности и прослеживаемости (traceability).*

## 📋 Проект 2: Мобильное приложение Яндекс.Метро (Android)
Тестирование новой сборки после рефакторинга и проведение полного регрессионного цикла.

### Выполненные задачи:
* **Функциональное тестирование:** проверка обновленной логики приложения согласно выделенным требованиям.
* **Регрессионное тестирование:** проверка стабильности текущих функций после внесения правок в код.
* **Mobile Specific Testing:** тестирование особенностей мобильной среды (переключение сетей 3G/LTE/Wi-Fi, работа в авиарежиме, входящие звонки, смена ориентации экрана, прерывания).
* **Тестирование на эмуляторе:** работа в Android Studio (конфигурация Honor 8, Android 9.0).
* **Баг-репортинг:** составление детальных отчетов о дефектах с указанием специфики мобильного окружения.

### Ссылки на артефакты:

### Инструментарий:
* **Android Studio (Emulator)**

## 📋 Проект 3: Веб-сервис Яндекс.Маршруты (Функционал каршеринга)
Тестирование нового модуля бронирования автомобилей: от проверки верстки по макетам до логики оплаты.

### Выполненные задачи:
* **Тестирование верстки (UI):** проверка соответствия макетам формы бронирования и интерактивных элементов карты.
* **Тест-дизайн сложных форм:** составление чек-листов для логики модальных окон «Способ оплаты» и «Добавление карты».
* **Функциональное тестирование:** написание детальных тест-кейсов для критического пути пользователя (кнопка «Забронировать»).
* **Кроссбраузерное тестирование:** проверка корректности отображения элементов в разных браузерах.
* **Баг-репортинг:** фиксация дефектов интерфейса и логики с подробными шагами воспроизведения.
